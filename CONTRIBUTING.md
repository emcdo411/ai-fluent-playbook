# Contributing to the AI-Fluent Playbook

## What Contributions Are Welcome

| Type | Description | Branch Prefix |
|------|------------|--------------|
| Capability Update | A tool or AI capability has materially shifted | capability/ |
| Empirical Addition | A citation or study that strengthens a claim | evidence/ |
| Framing Fix | A section that is accurate but missing structural context | framing/ |
| Typo or Clarity | Grammar or readability editing | fix/ |
| New Section | A topic gap identified by a reader | feature/ |

## Contribution Process

1. Fork this repository
2. Create a branch: git checkout -b capability/appendix-b-update
3. Make your changes in the relevant docs/ or playbook/ file
4. Update CHANGELOG.md under [Unreleased]
5. Open a Pull Request describing what changed and why

## Capability Update Standard

When submitting a capability update, include in the PR:
- Which AI generation your update reflects
- What specific playbook claim is now inaccurate
- What the corrected framing is
- Source or evidence for the capability shift
